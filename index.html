⚙ Reverse Engineering · Challenge #11

Stack Builder

The binary does not contain the flag as a visible string. It constructs the flag byte-by-byte on the stack at runtime. strings and xxd will find nothing useful.

Description

This binary builds its own flag from scratch at runtime. Instead of storing FlagVault{...} as a string constant in .rodata, the program assigns each character individually to local variables using integer literals and arithmetic expressions scattered across the function body.

The compiler emits a series of MOV BYTE PTR [RBP-N], 0xNN instructions — one per character. The complete flag is only visible as a contiguous string in memory at runtime, never in the binary's static data sections.

Your mission: catch the flag as it lives on the stack. Use GDB to break just before printf, or use Ghidra to collect and reconstruct the individual byte values from the disassembly.

Ghidra Static Reconstruction

Step 1 — Open the binary in Ghidra. Find build_and_print in the Functions list or Symbol Tree.

Step 2 — In the Decompiler panel, look for the series of local_XX = 0xNN assignments. Each is one character of the flag stored at a specific stack offset.

Step 3 — Note both the offset number (determines position) and the hex value (the character). Sort by offset ascending, then convert each hex byte to its ASCII character.

Quick Python check: bytes([0x46,0x6c,0x61,0x67,...,0x7d]).decode()
